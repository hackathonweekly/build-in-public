# Build In Public 活动组织者工作内容

本文档详细列出了活动组织者在 Build In Public 活动前、中、后的具体工作内容，以及飞书机器人开发任务。作为活动的核心推动者，组织者需要全面负责活动的策划、运营、技术支持和后续跟进。

## 一、活动前期准备

### 活动策划
- 确定活动主题、时间和周期（21天）
- 制定打卡规则和有效打卡标准
- 设计奖励机制和项目展示方式
- 安排双周线上会议的具体时间和流程

### 宣传推广
- 撰写微信公众号推文
- 设计海报和活动宣传图
- 在社区渠道发布活动信息
- 邀请往期参与者和潜在参与者

### 技术准备
- 创建飞书群组
- 开发飞书机器人（详见下方开发任务）
- 搭建项目展示网站页面
- 准备报名表单和资料

### 资料准备
- 编写活动指南文档
- 制作打卡模板和示例
- 准备欢迎信息和常见问题解答
- 设计目标设定接龙模板

## 二、活动进行中

### 日常管理
- 每日检查群内打卡情况
- 解答参与者问题
- 协调解决技术和沟通问题
- 确保飞书机器人正常运行

### 内容运营
- 组织双周线上会议（准备议程、主持会议）
- 每周五发布周末挑战主题
- 定期分享优质打卡案例
- 促进群内交流和项目匹配

### 技术支持
- 监控飞书机器人运行状态
- 解决打卡统计和提醒问题
- 优化机器人功能
- 处理数据统计和报告错误

### 社区维护
- 审核新成员加入
- 维护群内氛围
- 处理特殊情况和争议
- 鼓励互助和积极反馈

## 三、活动后期

### 数据统计
- 统计完成挑战的参与者
- 汇总所有项目信息和进展
- 分析活动参与度和成果
- 形成活动总结报告

### 成果展示
- 收集达标项目的展示材料
- 在网站发布项目展示
- 编写公众号优秀项目专题
- 组织优秀项目Demo日

### 反馈收集
- 向参与者发放活动反馈问卷
- 收集改进建议
- 总结经验和教训
- 制定下期活动改进计划

### 后续跟进
- 维护项目展示网站
- 关注优秀项目后续发展
- 建立往期参与者社区
- 为下期活动准备宣传案例

## 四、飞书机器人开发任务

### 基础功能开发
- 设计机器人交互流程
- 实现打卡识别与统计功能
- 开发数据存储和分析模块
- 设计用户交互界面和指令系统

### 打卡管理功能
- 开发打卡格式识别算法
- 实现多种打卡标签分类
- 设计打卡提醒系统
- 开发打卡数据存储和查询

### 报告生成功能
- 实现每日/周报告自动生成
- 设计个人统计查询接口
- 开发数据可视化模块
- 设计报告推送机制

### 社区功能开发
- 实现项目相似度匹配算法
- 开发问题解答路由功能
- 设计周末挑战推送系统
- 实现优秀项目推荐机制

### 技术维护
- 设计错误处理和日志系统
- 制定数据备份和恢复方案
- 优化机器人性能和响应速度
- 实施功能迭代和版本更新

## 五、时间规划表

| 时间段 | 主要工作内容 |
|--------|------------|
| 活动前2周 | 活动策划、宣传素材准备、文档编写 |
| 活动前1周 | 飞书群组创建、机器人部署、报名收集 |
| 活动第1天 | 开营仪式、目标设定指导、参与者注册 |
| 活动第1-7天 | 日常打卡管理、激励参与、问题解答 |
| 活动第7天 | 组织第一次线上会议、首周总结 |
| 活动第8-14天 | 继续日常管理、推送周末挑战、分享优质案例 |
| 活动第14-20天 | 项目进展跟进、准备结营、收集项目材料 |
| 活动第21天 | 组织第二次线上会议、结营仪式 |
| 活动后1周 | 数据统计、成果整理、反馈收集 |
| 活动后2周 | 项目展示发布、撰写总结报告、规划下期活动 |
