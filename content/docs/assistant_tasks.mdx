# 飞书小助手功能设计

## 基本信息

- **名称**：Build In Public 小助手
- **平台**：飞书机器人
- **用途**：辅助管理 Build In Public 21天独立开发者线上活动

## 核心功能

### 1. 打卡管理

- **打卡识别**
  - 自动识别包含 `#名字 #D数字 #进度` 格式的消息为有效打卡
  - 支持识别多种打卡标签：`#进度`、`#求助`、`#反馈`、`#资源`、`#灵感`

- **打卡统计**
  - 自动统计每位参与者的打卡次数
  - 记录每人首次打卡的日期作为起始日
  - 标记每条打卡消息的类型（进度、求助、反馈等）

- **打卡提醒**
  - 每晚8点检查当日未打卡的参与者并发送提醒
  - 对连续3天未打卡的参与者发送个人提醒

### 2. 数据统计与报告

- **每日报告**
  - 每天晚上10点生成当日打卡数据摘要
  - 包含：总打卡人数、新增打卡人数、热门话题/标签

- **周报告**
  - 每周日晚上生成周报告
  - 包含：本周活跃度排行、打卡率统计、参与者进度概览
  - 突出展示优质打卡内容（点赞数高的）

- **个人报告**
  - 参与者可通过 `@小助手 我的统计` 查询个人打卡情况
  - 显示累计打卡次数、连续打卡天数、距离目标还需打卡次数

### 3. 社区互动促进

- **项目相似度匹配**
  - 每周一次分析所有参与者的项目描述和打卡内容
  - 推送潜在的相似项目和可能的合作者名单
  - 示例：`@张三 @李四 你们的项目在"用户验证"方面有相似之处，建议互相交流`

- **问题解答路由**
  - 自动识别带有 `#求助` 标签的消息
  - 根据问题内容标签化（前端、后端、设计、营销等）
  - 在群内@可能帮助解决问题的其他成员

- **周末挑战**
  - 每周五发布一个共性问题挑战
  - 汇总相关打卡内容，生成主题讨论

### 4. 用户管理

- **新成员欢迎**
  - 欢迎新加入的成员，自动发送活动介绍和指南
  - 引导新成员设置个人项目信息和目标

- **参与者注册**
  - 通过 `#注册` 命令收集参与者基本信息
  - 格式：`#注册 项目名称:xxx 项目描述:xxx 目标:xxx`
  - 建立参与者数据库，用于后续统计和匹配

- **目标管理**
  - 记录每位参与者的目标设定
  - 提供 `@小助手 更新目标` 功能允许调整目标

### 5. 展示与激励

- **达标认证**
  - 自动识别达到9次有效打卡的参与者
  - 授予虚拟徽章并公开表彰

- **项目展示收集**
  - 活动结束前，提示达标参与者填写项目展示信息
  - 收集项目链接、截图和简介

- **优秀项目推荐**
  - 根据打卡质量、参与度和目标完成情况
  - 生成"优秀项目"候选名单

## 技术要求

- 自然语言处理能力，识别打卡内容和分类
- 数据存储与分析，追踪参与者活动情况
- 定时任务执行，发送提醒和报告
- 用户交互界面，响应指令和查询

## 实现优先级

1. **第一阶段（基础功能）**
   - 打卡识别与统计
   - 每日/周报告生成
   - 新成员欢迎与指引

2. **第二阶段（增强功能）**
   - 打卡提醒系统
   - 个人统计查询
   - 达标认证

3. **第三阶段（高级功能）**
   - 项目相似度匹配
   - 问题解答路由
   - 周末挑战自动化

## 使用场景示例

### 场景一：日常打卡
用户发送：`#张三 #D1 #进度 今天完成了登录页面设计...`
机器人：记录打卡，更新统计数据（无需回复）

### 场景二：查询个人情况
用户：`@小助手 我的统计`
机器人：`张三，你已完成4次打卡，连续打卡2天。距离目标还需5次打卡。`

### 场景三：求助标签
用户：`#张三 #D5 #求助 React Hook中遇到了闭包问题...`
机器人：`已收到求助，已通知可能帮助你的成员：@李四 @王五（他们最近讨论过相关话题）`

### 场景四：项目匹配
机器人：`@张三 @李四 我发现你们的项目都涉及到了"AI图像处理"，可能有合作空间。`

### 场景五：达标通知
机器人：`恭喜@张三 已完成9次有效打卡，达成Build In Public挑战！请填写项目展示信息：[表单链接]`
