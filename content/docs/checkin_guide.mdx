# 打卡指南

有效的打卡不仅是记录进度，更是一种反思和分享的过程。本指南帮助你了解如何进行高质量的打卡，获得最大收益。

## 打卡基本规则

- **打卡频率**：根据个人进度自由打卡，21天内至少9次有效打卡
- **打卡方式**：在飞书群内直接发送打卡消息
- **打卡时间**：建议每天固定时间打卡，养成习惯
- **提醒机制**：机器人每晚8点提醒未打卡的参与者

## 打卡格式

标准打卡格式如下：

```
#名字 #D1 #进度
1. 今日完成：[具体内容]
2. 遇到的问题：[问题描述]
3. 解决方法：[解决方案]
4. 明日计划：[计划内容]
5. [可选]截图或demo视频
```

示例：
```
#张三 #D5 #进度
1. 今日完成：完成了用户登录界面的UI设计，实现了表单验证
2. 遇到的问题：移动端适配出现错位
3. 解决方法：使用media query调整了布局
4. 明日计划：实现注册功能和邮箱验证
5. [附图]
```

## 打卡内容建议

### 🎯 什么是高质量打卡内容

1. **具体且量化**
   - ✅ "完成了5个API接口的开发和单元测试"
   - ❌ "写了一些代码"

2. **包含思考和反思**
   - ✅ "使用Redux管理状态后，发现组件耦合度降低了"
   - ❌ "实现了状态管理"

3. **分享遇到的问题和解决方案**
   - ✅ "遇到跨域问题，通过配置CORS解决"
   - ❌ "遇到了一些bug"

4. **有视觉呈现**
   - 截图、简短视频或GIF动画
   - 原型链接或可访问的demo

### 💡 打卡类型多样化

1. **进度打卡**：分享当天的开发进度和成果
2. **问题打卡**：分享遇到的技术难题，寻求帮助
3. **灵感打卡**：分享新的想法和创意
4. **用户反馈打卡**：分享收集到的用户反馈和计划改进
5. **里程碑打卡**：分享重要功能完成或版本发布

## 特殊标签使用

- **#求助**：当你遇到难以解决的问题时
- **#反馈**：当你希望获得他人对你项目的意见
- **#资源**：分享有用的开发资源
- **#灵感**：分享创意和想法

## 打卡技巧

1. **保持简洁**：控制打卡内容在300字以内
2. **重点突出**：将最重要的进展放在开头
3. **可视化**：尽可能附上截图或短视频
4. **诚实面对**：遇到困难时如实分享，寻求帮助
5. **定时打卡**：养成固定时间打卡的习惯

## 常见问题

### Q: 一天可以打卡多次吗？
A: 可以，但只计算一次有效打卡。

### Q: 有几天没打卡，还能继续参与吗？
A: 当然可以。活动要求21天内至少9次有效打卡，并没有要求连续打卡。

### Q: 如何确认我的打卡已被记录？
A: 飞书机器人会自动统计打卡数据，每周会推送一次打卡统计报告。

### Q: 我的项目进展很小，值得打卡吗？
A: 绝对值得！小进展也是进展，分享具体的思考过程和遇到的问题同样有价值。
